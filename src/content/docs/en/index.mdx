The Varg language spec is split between target design and the currently executable MVP. This manual prioritizes the executable scripting subset; `.vscene`, `.vmodel`, `.vasset`, and future APIs are marked where relevant.
:::

## Recommended learning path

### Path A: First session

1. Read [Getting Started](/en/getting-started/) to find the editor and sample projects.
2. If you have no programming background, start with [Programming Primer](/en/scripting/programming-primer/).
3. Read [Scripting Basics](/en/scripting/basics/) and [Lifecycle and Input](/en/scripting/lifecycle-input/).
4. In [Control Flow and Waiting](/en/scripting/control-flow/), learn `if`, timers, and one simple loop.
5. Read [Organizing Gameplay Scripts](/en/scripting/gameplay-loop-patterns/) to structure a clear gameplay loop.

### Path B: Build gameplay

1. Read [State, Variables, and Exports](/en/scripting/state-export/) to separate tuning parameters from runtime state.
2. If you are not sure how to organize a full `update()`, read [Organizing Gameplay Scripts](/en/scripting/gameplay-loop-patterns/).
3. Follow [Intermediate Exercise: Collection Loop](/en/tutorials/first-playable-loop/).
4. Browse [Example Walkthroughs](/en/scripting/examples/) and understand one small script first.
5. Follow [Build an Endless Jumper](/en/tutorials/jump-jump/).

### Path C: Expand and finish

1. Read [Advanced Runtime APIs](/en/scripting/advanced-runtime-apis/) to connect scene, UI, audio, mouse, and rendering commands.
2. Then build [Build a Shooting Range](/en/tutorials/fps-arena/) to practice multi-system state.
3. Finish with [Editor Workflow](/en/features/editor-workflow/), [Assets and Scenes](/en/features/assets-scenes/), and [Build and Package](/en/features/build-package/).
