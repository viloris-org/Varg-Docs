:::note[当前状态]
Varg 的语言规范分为目标设计和当前可执行 MVP。本手册的脚本教程优先使用当前可执行子集；涉及 `.vscene`、`.vmodel`、`.vasset` 和未来 API 的地方会明确标注。
:::

## 推荐学习路径

### 路线 A：第一次上手

1. 读 [快速上手](/getting-started/)，确认编辑器和示例项目在哪里。
2. 如果你没有编程经验，先读 [零基础脚本入门](/scripting/programming-primer/)，理解变量、函数、条件和每帧更新。
3. 读 [脚本基础](/scripting/basics/) 和 [生命周期与输入](/scripting/lifecycle-input/)，理解脚本挂在哪里、什么时候运行、怎么读输入。
4. 在 [控制流与等待](/scripting/control-flow/) 里只掌握 `if`、计时器和一个简单循环。`wait()` 先知道有这个工具即可。

### 路线 B：开始做玩法

1. 读 [状态、变量与导出](/scripting/state-export/)，把“调参”和“运行时状态”分清楚。
2. 跟做 [中间练习：做一个可玩的收集循环](/tutorials/first-playable-loop/)，先把移动、拾取、得分、倒计时和 HUD 串起来。
3. 看 [示例拆解](/scripting/examples/)，先挑一个小脚本读懂，不用一次读完。
4. 跟做 [制作无尽跳跃](/tutorials/jump-jump/)。它比射击训练场更适合作为第一个完整项目。

### 路线 C：扩展和收尾

1. 读 [高级运行时 API](/scripting/advanced-runtime-apis/)，把场景、UI、音频、鼠标和渲染命令串进玩法循环。
2. 再做 [制作射击训练场](/tutorials/fps-arena/)，练习多系统状态：移动、武器、刷怪、胜负和 HUD。
3. 最后看 [编辑器工作流](/features/editor-workflow/)、[资源与场景](/features/assets-scenes/) 和 [构建打包](/features/build-package/)。
