このページで扱うのは「エディターを開いた後にどう作業するか」です。Varg エディターの内部アーキテクチャではありません。

通常、エディターでは 3 つのことを行います。プロジェクトを管理し、シーンを整理し、Play モードでスクリプトを検証します。スクリプト、シーン、アセットは普通のプロジェクトファイルのままです。エディターは、それらのファイルを見やすく、変更しやすく、テストしやすくするためのものです。

## 基本流れ

1. Varg を起動し、Hub に入ります。
2. 新しいプロジェクトを作成するか、既存のプロジェクトディレクトリを開きます。
3. シーンを開き、Hierarchy でオブジェクトを選択します。
4. Inspector で位置、回転、スケール、コンポーネントパラメータを調整します。
5. Scene View で 3D レイアウトを確認します。
6. Play をクリックし、Game View で物理とスクリプトを実行します。
7. Stop 後に編集状態へ戻り、調整を続けます。

編集状態と実行状態は分けて理解してください。編集状態で変更するのはシーンファイルです。Play モードで動くのはランタイム世界のコピーです。そのため、スクリプトを試しても、失敗した実行がシーンファイルを壊す心配を減らせます。

## Hub の役割

Hub はプロジェクト入口です。プロジェクトの作成、プロジェクトの打开、最近使ったプロジェクトへ戻ることを担当します。

プロジェクトには少なくとも次が必要です。

- `Varg.toml`: プロジェクトマニフェスト。既定シーンとスクリプトディレクトリを宣言します。
- `scenes/`: シーンファイル。
- `scripts/`: `.varg` スクリプト。
- `assets/`: モデル、テクスチャ、音声などのアセット。

まず完全なプロジェクトを見たいだけなら、サンプルを直接開けます。

- [examples/project/jump_jump](https://github.com/viloris-org/Varg/tree/main/examples/project/jump_jump)
- [examples/project/fps_arena](https://github.com/viloris-org/Varg/tree/main/examples/project/fps_arena)

## よく使うコンポーネント

最初から全コンポーネントを覚える必要はありません。まず次を押さえてください。

| コンポーネント | 用途 |
| --- | --- |
| Camera | ゲーム視点を定義する |
| Light | シーン照明 |
| MeshRenderer | モデルまたはメッシュを表示する |
| Rigidbody | 物理シミュレーションに参加する |
| Collider | 衝突形状 |
| Script | `.varg` スクリプトを取り付ける |

動く最小シーンには通常、Camera、Light、そしてスクリプト付きまたは対話可能なエンティティが少なくとも 1 つ必要です。

## スクリプトをシーンに入れる

スクリプトは自動では実行されません。必ずシーンオブジェクトに取り付ける必要があります。典型的な流れは次の通りです。

1. プロジェクト内で `.varg` ファイルを作成または導入します。
2. 対象エンティティを選択します。
3. Script コンポーネントを追加します。
4. `scripts/player_controller.varg` のようなスクリプトリソースをバインドします。
5. Inspector で `@export var` が公開したパラメータを調整します。
6. Play モードで挙動を検証します。

例:

```swift
script JumpPlayer {
    source: "scripts/jump_player.varg"
    maxCharge: 1.25
    jumpScale: 5.0
}
```

ここで `source` はスクリプトファイルを指します。下のフィールドはスクリプト内のエクスポートパラメータを上書きします。つまり、スクリプトはルールを担当し、Inspector または `.vscene` は調整値を担当します。

## Play モードで見ること

Play モードに入ったら、まず 4 つを確認します。

- 入力がスクリプトに届いているか。
- オブジェクト位置と回転が期待通り変化するか。
- HUD またはログが現在状態を説明できるか。
- Stop 後にきれいに編集状態へ戻れるか。

最初から全システムをつながないでください。まず 1 つのオブジェクトを動かし、その後で衝突、UI、効果音、生成ロジックを足します。問題が出た時の範囲が小さくなります。

## ファイルとの関係

Varg プロジェクトはエディター内でしか変更できないわけではありません。エディターでオブジェクトやパラメータを調整してもよいですし、プロジェクトファイルを直接編集してもかまいません。

よくある役割分担:

- `.vscene` はシーン構造、オブジェクト、コンポーネント、スクリプト取り付け関係を書く。
- `.varg` は入力、タイマー、状態機械、UI などのランタイムロジックを書く。
- `.vasset` はアセット登録とインポート設定を書く。
- `Varg.toml` はプロジェクト入口と既定シーンを書く。

エディターは空間関係の確認とパラメータ調整に向いています。テキストファイルは review、サンプルのコピー、一括変更、ツールによる初稿生成に向いています。両者は衝突しません。重要なのは、プロジェクトファイルを常に読みやすく保つことです。
