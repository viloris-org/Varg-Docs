title: 状態、変数、エクスポート
description: 調整可能パラメータ、永続状態、ローカル変数を区別します。
---

Varg スクリプトは明示的な意図を重視します。変数宣言の場所を見るだけで、それが設定なのか、状態なのか、一時的な計算値なのかが分かるべきです。

プログラミング未経験なら、変数を「スクリプトの小さなメモ帳」と考えてください。一部はレベル制作者が変更する内容、一部はスクリプト自身が覚える内容、一部はこのフレームだけの一時計算です。

## 3 種類の変数

| 書き方 | ライフサイクル | 用途 |
| --- | --- | --- |
| `@export var` | スクリプトと一緒に永続し、エディターで調整できる | 速度、ダメージ、クールダウン、数量 |
| スクリプトスコープの `var` | スクリプトと一緒に永続する | 弾薬、タイマー、段階、初期化済みか |
| 関数内の `let` / `var` | その関数呼び出し内だけ有効 | 一時計算、ループカウンタ、局所結果 |

## エクスポートパラメータ

エクスポートパラメータは「外から調整できるつまみ」です。移動速度、ダメージ、クールダウン時間などが該当します。通常、プレイヤーが実行中に自分で変更する状態ではなく、制作者が手触りを調整する時に変更する値です。

```swift
script WeaponCooldown {
    @export var fireRate: Float = 0.5
    @export var damage: Int = 10

    var canFire: Bool = true
    var ammo: Int = 30
}
```

`fireRate` と `damage` はデザイナーまたはレベル制作者が調整するパラメータです。`canFire` と `ammo` はランタイム内部状態です。

## 永続状態

永続状態は「スクリプト自身が覚えること」です。残り弾薬、リロード中かどうか、何秒経過したか、などです。

スクリプトスコープの `var` はフレームをまたいで保持されます。

```swift
script Timer {
    var elapsed: Float = 0.0

    func update(_ dt: Float) {
        elapsed += dt
    }
}
```

`elapsed` を `update()` の内部に書くと、毎フレーム作り直されるため、累積タイマーには向きません。

## ローカル変数

ローカル変数は「この小さなコード片で一時的に計算した結果」です。長期記憶ではなく、現在の計算を明確にするために使います。

```swift
func update(_ dt: Float) {
    let pulse: Float = sin(Time.time * 3.0) * 0.2
    position.y = 1.0 + pulse
}
```

`pulse` はこのフレームの計算結果だけなので、`let` で十分です。

## 旧式の `state.name`

現在の MVP はまだ次を受け付ける可能性があります。

```swift
state.ammo -= 1
```

新しいスクリプトでは、宣言的な永続状態として書くことを優先してください。

```swift
var ammo: Int = 30

func update(_ dt: Float) {
    ammo -= 1
}
```

この方が、エディター、検証器、AI エージェントに理解されやすくなります。
